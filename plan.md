# 🧾 Obsidian家計簿（レシート画像認識）構築プラン

家計簿の自動化（レシート画像認識 ➔ カテゴリ分類 ➔ データベース化 ➔ Obsidianテーブル表示）についての検討プランです。

---

## 1. AI（モデル）の選定・比較

レシートの画像認識（OCR + 構造化データの抽出）を行うにあたり、**ローカルLLM**と**クラウドLLM（API）**の2つのアプローチが考えられます。

| 項目 | ローカルLLM (Ollama等) | クラウドLLM (Gemini API 等) |
| :--- | :--- | :--- |
| **代表的なモデル** | `qwen2-vl`, `llama3.2-vision` | `Gemini 1.5/2.5 Flash` |
| **導入の容易さ** | Ollamaのインストールが必要。モデルのダウンロードに時間がかかる。 | APIキーの取得のみで、すぐに使える。 |
| **日本語レシート精度** | 漢字や歪み、細かい文字の認識精度は中位。 | **極めて高い**。潰れた文字や歪みも高精度で読み取る。 |
| **レスポンス速度** | ローカルPCのGPU性能に依存（数秒〜数十秒）。 | **高速**（通常1〜3秒程度）。 |
| **ランニングコスト** | **完全無料**（電気代のみ）。 | ほぼ無料（無料枠が非常に広い、超えても1枚あたり数銭）。 |
| **プライバシー** | **完璧**（データが外部に一切送信されない）。 | 送信される（オプトアウト設定は可能だが気になる場合は注意）。 |
| **動作環境** | 比較的高いPCスペック（GPUメモリ 8GB以上推奨）が必要。 | スペック不問（スマホやノートPCでも動作）。 |

### 💡 結論としてどちらが「楽」か？
- **手軽さと精度を重視するなら「クラウドAPI（Gemini Flash）」が圧倒的に楽**です。特にレシートはフォントが細かくレイアウトが複雑なため、ローカルVLM（視覚言語モデル）だと読み取りエラーやカテゴリ誤分類が起きやすく、修正コードの作成やモデル調整の手間が発生しがちです。
- **プライバシーを最優先し、お金のデータを絶対に外に出したくないなら「ローカルLLM（Ollama + Qwen2-VL）」**になります。Ollamaを使えば、Pythonから数行で呼び出す環境は簡単に作れます。

> [!TIP]
> **推奨アプローチ**: 
> まずは最も開発が簡単な **Gemini API** を使って「画像読み取り ➔ Obsidian保存 ➔ テーブル表示」の全体の流れ（パイプライン）を構築し、動くことが確認できてから、必要に応じてAPI部分を **Ollama (ローカルLLM)** に差し替えるのが、手戻りが少なく最もスムーズです。

## 2. システム構成イメージ

以下のような構成でシステムを構築します。

```mermaid
graph TD
    A[レシート画像を撮影・保存] --> B[Python/Node.js スクリプト実行]
    B --> C{AIによる解析}
    C -->|画像データを送信| D[ローカルLLM (Ollama) / Gemini API]
    D -->|認識結果 (JSON形式)| C
    C --> E[Obsidian VaultへMarkdown出力]
    E --> F[Obsidian Dataviewで自動テーブル表示]
```

### Obsidianでのデータ保持と表示方法
1. **Markdownファイルによる個別管理（推奨）**
   - レシート1枚につき1つのMarkdownファイルを生成します（例: `acountbook/data/2026-06-27_スーパーA.md`）。
   - ファイルの先頭（Frontmatter）に、AIが抽出したデータを書き込みます。
     ```yaml
     ---
     date: 2026-06-27
     shop: スーパーA
     total: 1280
     items:
       - name: 牛乳
         price: 220
         category: 食費
       - name: 洗剤
         price: 350
         category: 日用品
     ---
     ```
2. **Dataviewプラグインによるテーブル表示**
   - Obsidian側で `Dataview` プラグインを使用し、以下のようなクエリを書くだけで、自動的に綺麗な一覧テーブルと合計金額が集計されます。
     ```sql
     TABLE date as 日付, shop as 店舗, total as 金額
     FROM "acountbook/data"
     SORT date DESC
     ```
